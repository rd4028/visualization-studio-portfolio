## OWASP Top 10 Study Notes

| Risk Area | What It Means | Practical Relevance |
| --- | --- | --- |
| Broken access control | Users can access resources they should not access | Important for protecting customer and business data |
| Cryptographic failures | Sensitive data is not protected properly | Important for privacy and compliance |
| Injection | Untrusted input affects commands or queries | Important for secure application design |
| Insecure design | Security is not considered early enough | Important for process and system improvement |
| Security misconfiguration | Systems are exposed due to weak settings | Important for IT support and operations |

## Authentication Concepts

Authentication verifies who a user is. Authorisation controls what that user can access. In business systems, both are important for protecting customer information, internal data, and operational systems.

## Practical Security Checklist

- Use strong, unique passwords.
- Enable multi-factor authentication where possible.
- Apply least-privilege access.
- Review permissions regularly.
- Avoid exposing sensitive data in files, screenshots, or repositories.
- Document security steps clearly so they can be repeated.
