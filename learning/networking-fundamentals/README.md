- Basic troubleshooting workflow

## Core Concepts

| Concept | Meaning | Practical Relevance |
| --- | --- | --- |
| TCP/IP | Protocol suite used for network communication | Helps explain how systems communicate |
| DNS | Translates names into IP addresses | Useful when troubleshooting website or service access |
| HTTP/HTTPS | Protocols used for web communication | Important for web systems and secure access |
| OSI model | Layered model for understanding network behaviour | Helps structure troubleshooting |
| IP addressing | Identifies devices on networks | Important for connectivity and support tasks |

## Troubleshooting Workflow

1. Confirm the problem and scope.
2. Check whether the issue affects one user, one device, or many users.
3. Verify physical or wireless connectivity.
4. Check IP configuration.
5. Test DNS resolution.
6. Test access to the target service.
7. Document findings and escalation steps.
